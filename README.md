# QQQ 시험·인증 툴

온라인 연결 없이 로컬 환경에서 검사원 자격 인증 평가를 진행하고, 결과 원본과 인증서 이력을 SQLite로 관리하는 **검사원 시험·자격 인증 관리 시스템**입니다.

## 제품 정의

이 도구는 단순 필기시험 응시 도구가 아니라 `검사원 자격 인증을 위한 다중 평가·검증 관리 시스템`입니다. 기존 필기시험 MVP를 기반으로 다음 평가와 증빙을 하나의 로컬 흐름으로 통합합니다.

- 필기시험(`written_exam`)
- 계수형 Gage R&R(`attribute_rr`)
- 계량형 Gage R&R(`variable_rr`)
- 교육 이수 검증(`training_verification`)
- 자격 판정과 인증서 발행
- SQLite 원본 저장, 로컬 조회, CSV 내보내기

## 핵심 설계 원칙

- **SQLite 원본 저장소:** 제출 답안, 판정값, 측정값, 산출 결과, 자격 판정, 인증서 이력의 원본은 `data/exam_results.sqlite3`에 저장합니다.
- **CSV 보조 기능:** CSV는 원본 저장소가 아니라 내보내기, 백업, 외부 제출용 보조 형식입니다. CSV import가 필요한 계량형 R&R도 확정 원본은 SQLite에 저장합니다.
- **불변 결과:** 제출 또는 확정 이후 원본 결과는 수정하지 않으며, 변경은 취소·재평가·재발행 이력으로 관리합니다.
- **평가 유형 고정:** 지원 평가 유형은 `written_exam`, `attribute_rr`, `variable_rr`, `training_verification` 네 가지입니다.
- **사진 기반 판정 흡수:** 사진 기반 샘플 판정은 독립 시험이 아니라 `attribute_rr`의 `sampleMode=image`입니다. `image_judgment` 독립 평가 유형은 만들지 않습니다.
- **인증서 발행 통제:** 인증서는 필수 평가와 필수 증빙이 모두 충족되고 `certification_decision=approved`인 경우에만 시스템 확정값으로 발행합니다. 점수, 등급, 평가 항목명은 수기 입력하지 않습니다.
- **오프라인 우선:** 온라인 서버, 계정, 실시간 동기화, 실시간 감독 기능은 추가하지 않습니다.
- **측정 기반 최적화:** 실제 병목이 확인되기 전에는 캐시, 병렬화, 대형 구조 변경을 하지 않습니다.

## 자격 인증 흐름

```text
응시자 등록
  ↓
자격 유형 선택
  ↓
필수 평가 계획 확인
  ↓
필기시험 / 계수형 R&R / 계량형 R&R / 교육 이수 확인
  ↓
자격 판정
  ↓
인증서 발행
  ↓
SQLite 원본 저장 / 로컬 조회 / CSV 내보내기
```

## 평가 유형

| 평가 유형 | 목적 | 주요 산출물 |
| --- | --- | --- |
| `written_exam` | 검사 기준·절차·부적합 판정 기준 이해도 확인 | 총점, 문항별 정오, 과락 여부, 합격 여부, 등급 |
| `attribute_rr` | OK/NG 판정 일치성, 반복성, 오류율 확인 | 전체 일치율, 반복 일치율, 1종 오류율, 2종 오류율, 불량 유형 일치율, 최종 판정 |
| `variable_rr` | 계측기 사용과 수치 측정 능력 확인 | EV, AV, GRR, %GRR, Part Variation, Total Variation, ndc, 최종 판정 |
| `training_verification` | 필수 교육 이수, 최소 교육 시간, 증빙 확인 | 교육 충족 여부, 시간 충족 여부, 증빙 등록 여부 |

계수형 R&R의 `sampleMode`는 `image`, `physical_sample`, `mixed`로 관리합니다. 사진 파일 경로와 SHA-256 해시는 샘플 무결성 확인용이며, 기준 OK/NG와 기준 설명은 제출 전 응시 화면에 노출하지 않습니다.

## 현재 구현 상태와 이번 PR 범위

현재 저장소에는 외부 패키지 없이 실행하는 로컬 웹 MVP의 필기시험 수직 슬라이스가 포함되어 있습니다. 이번 작업은 구현 코드를 수정하지 않고 문서의 제품 정의와 저장 구조 설명을 먼저 정리합니다.

1차 작업 범위는 문서 정합성 정리입니다. DB 스키마 초안, 계수형 R&R, 계량형 R&R, 인증서 발행 검증 구현은 후속 PR에서 순차적으로 진행합니다.

## 로컬 실행

개발 검증은 Node.js 20 이상과 Python 3를 기준으로 합니다.

```bash
npm test
npm run check
npm start
```

서버 실행 후 브라우저에서 `http://localhost:4173`을 열어 JSON 시험지를 선택합니다. 현재 구현의 필기시험 결과 저장 동작은 후속 SQLite 전환 작업에서 문서 기준과 맞추어 정리합니다.
