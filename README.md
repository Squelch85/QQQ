# QQQ 시험 응시 툴

안정적인 온라인 시험 응시 경험을 제공하기 위한 웹 기반 시험 응시 도구입니다.

## 제품 목표

- 네트워크 장애와 새로고침 상황에서도 응시자의 답안을 보존합니다.
- 시험 시간과 제출 여부는 서버를 기준으로 일관되게 판정합니다.
- 최소 권한과 데이터 최소화 원칙으로 시험 정보와 개인정보를 보호합니다.
- 측정된 병목만 최적화하고, 재현 가능한 품질 지표로 출시 여부를 판단합니다.

## MVP 범위

1. 응시자 인증 및 시험 입장
2. 객관식 단일·복수 선택, 단답형 문제 응시
3. 서버 기준 남은 시간 표시
4. 답안 자동 저장 및 재접속 복구
5. 중복되지 않는 최종 제출과 제출 결과 확인
6. 관리자용 시험 등록 및 응시 현황 확인

웹캠 감독, 화면 녹화, AI 기반 부정행위 탐지는 개인정보와 운영 정책 검토가 선행되어야 하므로 MVP에서 제외합니다.

## 핵심 설계 원칙

- **서버 권위:** 시험 시작, 마감, 제출 가능 여부는 서버가 판정합니다.
- **답안 보존:** 답안은 자동 저장하며 재시도 요청은 멱등하게 처리합니다.
- **명시적 상태 전이:** 응시 세션의 허용된 상태 전이만 수행합니다.
- **보안 경계:** 정답과 채점 기준은 응시자 API로 전달하지 않습니다.
- **점진적 확장:** 모듈형 모놀리스로 시작하고 실측된 필요가 있을 때만 인프라를 분리합니다.
- **측정 기반 최적화:** 성능 변경은 5회 이상 측정한 중앙값과 IQR로 검증합니다.

## 권장 구현 순서

1. 제품 정책과 상태 전이를 확정합니다.
2. API 계약과 데이터베이스 스키마를 작성합니다.
3. 답안 자동 저장과 최종 제출의 수직 슬라이스를 구현합니다.
4. 재접속, 만료, 중복 요청 및 장애 시나리오를 검증합니다.
5. 관리자 기능과 운영 관측성을 추가합니다.
6. 부하 시험으로 확인된 병목만 개선합니다.

## 현재 상태

현재 저장소는 **제작 방향과 품질 기준을 수립한 기획 단계**입니다. 기술 스택과 배포 환경은 구현 착수 전에 팀 역량, 운영 환경, 예상 동시 응시자 수를 기준으로 ADR(Architecture Decision Record)에서 확정합니다.
