# QQQ 시험 응시 툴

온라인 연결이나 서버 동기화 없이, 준비된 시험지를 열어 응시하고 즉시 평가·채점하는 **단순 시험 응시 도구**입니다.

## 제품 목표

- 네트워크, 계정, 서버 운영 없이 단일 환경에서 시험을 진행합니다.
- 응시자는 문제를 풀고 제출하면 즉시 채점 결과를 확인합니다.
- 시험 준비자는 시험지와 정답·배점만 준비해 동일한 채점 기준을 반복 적용합니다.
- 개인정보와 운영 복잡도를 최소화하고, 필요한 경우 결과 파일만 보관합니다.
- 측정된 병목만 최적화하고, 문서와 구현은 단순성을 우선합니다.

## MVP 범위

1. 시험지 파일 불러오기
2. 객관식 단일·복수 선택, 단답형 문제 응시
3. 로컬 타이머와 제출 전 미응답 문항 확인
4. 제출 시 자동 평가 및 점수 계산
5. 정답, 오답, 문항별 배점, 총점 표시
6. 선택적 결과 파일 저장 또는 출력

온라인 인증, 서버 저장, 자동 동기화, 재접속 복구, 관리자 대시보드, 실시간 감독 기능은 MVP에서 제외합니다.

## 핵심 설계 원칙

- **오프라인 우선:** 시험 응시, 평가, 채점은 네트워크 없이 완료되어야 합니다.
- **단순 제출:** 제출은 현재 화면 답안을 기준으로 한 번 채점하며 별도 서버 상태를 요구하지 않습니다.
- **채점 기준 분리:** 정답과 배점은 시험지 정의에 포함하되 응시 화면에서는 제출 전까지 노출하지 않습니다.
- **명시적 결과:** 총점, 문항별 정오, 획득 점수, 필요 시 해설을 즉시 표시합니다.
- **최소 저장:** 기본 동작은 메모리 기반이며, 결과 보관은 사용자가 선택한 파일 저장으로 제한합니다.
- **측정 기반 최적화:** 성능 변경은 5회 이상 측정한 중앙값과 IQR로 검증합니다.

## 권장 구현 순서

1. 시험지 파일 형식과 채점 규칙을 확정합니다.
2. 문제 표시, 답안 입력, 제출 버튼의 수직 슬라이스를 구현합니다.
3. 객관식·단답형 자동 채점과 결과 화면을 구현합니다.
4. 로컬 타이머, 미응답 확인, 결과 저장을 추가합니다.
5. 접근성, 오류 메시지, 파일 검증 테스트를 보강합니다.
6. 대표 시험지로 확인된 병목만 개선합니다.

## 현재 상태

현재 저장소에는 **외부 패키지 없이 실행하는 로컬 웹 MVP의 첫 수직 슬라이스**가 구현되어 있습니다. JSON 시험지 검증, 세 가지 문제 유형 응답, 메모리 기반 제출 스냅샷, 자동 채점, 결과 저장과 인쇄를 지원합니다. 기술 선택은 ADR에 기록하며 데스크톱 패키징과 추가 운영체제 지원은 실제 배포 요구가 확인될 때 결정합니다.

## 로컬 실행

초기 구현은 외부 패키지 설치가 필요 없는 로컬 웹 애플리케이션입니다. 응시만 할 때는 폴더를 원하는 위치로 복사한 뒤 `index.html`을 직접 열 수 있습니다. 개발 검증은 Node.js 20 이상과 Python 3가 설치된 환경을 기준으로 합니다.

```bash
npm test
npm run check
npm start
```

정적 서버를 사용할 때는 브라우저에서 `http://localhost:4173`을 열고, 파일을 직접 열 때는 복사한 폴더의 `index.html`을 열어 JSON 시험지를 선택합니다. 바로 확인하려면 [`examples/sample-exam.json`](examples/sample-exam.json)을 사용할 수 있습니다. 애플리케이션은 실행 중 네트워크 요청을 만들지 않으며 답안과 결과는 기본적으로 메모리에만 유지합니다.

## 시험지 형식

시험지는 `schemaVersion: 1`인 JSON 객체입니다. 현재 지원 유형은 다음과 같습니다.

- `single_choice`: `choices`와 `scoring.correctChoiceId` 사용
- `multiple_choice`: `choices`와 `scoring.correctChoiceIds` 사용, 정답 집합 완전 일치 시 득점
- `short_answer`: `scoring.acceptedAnswers`와 선택적 `normalization` 사용

단답형 정규화 옵션은 `trim`, `collapseWhitespace`, `caseInsensitive`이며 각 옵션을 명시적으로 `true`로 지정한 경우에만 적용합니다. 전체 예시는 예제 시험지에서 확인할 수 있습니다.

기술 선택 배경과 제약은 [`docs/adr/0001-local-web-es-modules.md`](docs/adr/0001-local-web-es-modules.md)에 기록했습니다.

## 검사원 평가 시험과 문항 변환

첫 화면의 **기본 시험 시작** 버튼으로 40문항의 `검사원 평가 시험`을 바로 실행할 수 있습니다. 이 기본 시험은 `src/standalone.js`에 포함되어 있어 저장소 밖으로 폴더를 옮긴 뒤 `index.html`을 직접 열어도 별도 경로 탐색 없이 시작됩니다. 시험 시작 전 성명, 사번, 부서를 입력하며, 제출할 때마다 응시자별 결과가 같은 브라우저의 로컬 저장소에 자동 누적됩니다. 전체 인원의 응시가 끝나면 **전체 리포트 저장**으로 JSON 파일을 보관하고, 첫 화면의 **리포트 불러오기**에서 평균 점수·합격률·오답률 상위 문항과 개인별 결과를 다시 확인할 수 있습니다.

**문항 JSON 변환**에서는 스프레드시트의 `문항번호`, `질문`, `정답` 세 열을 그대로 붙여 넣고, 지정한 출제 문항 수만큼 Fisher–Yates 방식으로 무작위 추출해 시험지 JSON으로 변환합니다. 선택된 문항은 합계가 정확히 100점이 되도록 0.01점 단위에서 최대한 균등하게 배점됩니다. 질문 셀 안의 선택지는 `1)`, `2)`, `3)`, `4)` 형식으로 작성해야 하며 여러 줄 셀과 큰따옴표로 감싼 TSV 입력을 지원합니다. 기본 시험 JSON은 [`examples/inspector-evaluation-exam.json`](examples/inspector-evaluation-exam.json)에서 내려받을 수 있습니다.
