# QQQ 시험 응시 툴

온라인 연결이나 서버 동기화 없이, 준비된 시험지를 열어 응시하고 즉시 평가·채점하는 **단순 시험 응시 도구**입니다.

## 제품 목표

- 네트워크, 계정, 서버 운영 없이 단일 환경에서 시험을 진행합니다.
- 응시자는 문제를 풀고 제출하면 즉시 채점 결과를 확인합니다.
- 시험 준비자는 시험지와 정답·배점만 준비해 동일한 채점 기준을 반복 적용합니다.
- 개인정보와 운영 복잡도를 최소화하고, 필요한 경우 결과 파일만 보관합니다.
- 측정된 병목만 최적화하고, 문서와 구현은 단순성을 우선합니다.

## MVP 범위

1. 시험지 파일 불러오기
2. 객관식 단일·복수 선택, 단답형 문제 응시
3. 로컬 타이머와 제출 전 미응답 문항 확인
4. 제출 시 자동 평가 및 점수 계산
5. 정답, 오답, 문항별 배점, 총점 표시
6. 선택적 결과 파일 저장 또는 출력

온라인 인증, 서버 저장, 자동 동기화, 재접속 복구, 관리자 대시보드, 실시간 감독 기능은 MVP에서 제외합니다.

## 핵심 설계 원칙

- **오프라인 우선:** 시험 응시, 평가, 채점은 네트워크 없이 완료되어야 합니다.
- **단순 제출:** 제출은 현재 화면 답안을 기준으로 한 번 채점하며 별도 서버 상태를 요구하지 않습니다.
- **채점 기준 분리:** 정답과 배점은 시험지 정의에 포함하되 응시 화면에서는 제출 전까지 노출하지 않습니다.
- **명시적 결과:** 총점, 문항별 정오, 획득 점수, 필요 시 해설을 즉시 표시합니다.
- **최소 저장:** 기본 동작은 메모리 기반이며, 결과 보관은 사용자가 선택한 파일 저장으로 제한합니다.
- **측정 기반 최적화:** 성능 변경은 5회 이상 측정한 중앙값과 IQR로 검증합니다.

## 권장 구현 순서

1. 시험지 파일 형식과 채점 규칙을 확정합니다.
2. 문제 표시, 답안 입력, 제출 버튼의 수직 슬라이스를 구현합니다.
3. 객관식·단답형 자동 채점과 결과 화면을 구현합니다.
4. 로컬 타이머, 미응답 확인, 결과 저장을 추가합니다.
5. 접근성, 오류 메시지, 파일 검증 테스트를 보강합니다.
6. 대표 시험지로 확인된 병목만 개선합니다.

## 현재 상태

현재 저장소는 **단순 오프라인 시험 응시·평가·채점 방향과 품질 기준을 수립한 기획 단계**입니다. 기술 스택은 구현 착수 전에 단일 실행 방식, 지원 운영체제, 시험지 파일 형식을 기준으로 ADR(Architecture Decision Record)에서 확정합니다.
