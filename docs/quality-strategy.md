# 품질 및 성능 전략

## 1. 품질 목표

우선순위는 **자격 판정 정확성 > 원본 결과 불변성 > 인증서 발행 통제 > 개인정보 최소화 > 사용 단순성 > 성능** 순서로 둔다. 성능 향상을 위해 평가 결과의 정확성, 결과 불변성, 발행 차단 조건을 희생하지 않는다.

## 2. 단위 테스트 범위

- 시험지 스키마와 필수 필드 검증
- 제출 후 답안 변경 차단
- 동일 입력 동일 채점 결과
- 필기시험 점수와 항목별 과락 판정
- 계수형 R&R 전체 일치율, OK 일치율, NG 검출률 계산
- 계수형 R&R 1종 오류율, 2종 오류율, 반복 일치율, 불량 유형 일치율 계산
- 계량형 R&R 측정값 누락과 숫자 형식 오류 검증
- 계량형 R&R EV, AV, GRR, %GRR, ndc 계산
- 교육 이수 시간과 증빙 충족 여부 판정
- 인증서 발행 가능 여부 판정

## 3. 통합 테스트 범위

- 필기시험 제출 → SQLite 저장 → 채점 결과 저장
- 계수형 R&R 제출 → 결과 계산 → SQLite 저장
- 계량형 R&R CSV import → 검증 → 계산 → SQLite 저장
- 교육 기록 등록 → 자격 판정 반영
- 필수 평가 누락 → 인증서 발행 차단
- 승인 완료 → 인증서 발행과 해시 저장
- 인증서 재발행 → 기존 이력 보존
- CSV export가 SQLite 원본에서 파생되는지 확인

## 4. E2E 테스트 범위

- 외관검사자 자격 인증 전체 흐름: 필기시험, 사진 기반 계수형 R&R, 교육 이수, 자격 판정, 인증서 발행
- 수입검사자 자격 인증 전체 흐름: 필기시험, 계량형 R&R, 교육 이수, 자격 판정, 인증서 발행
- 필수 R&R 누락 상태에서 인증서 발행 차단
- 교육 증빙 누락 상태에서 인증서 발행 차단
- 인증서 발행 후 응시자별, 자격별, 인증서별 조회

## 6. 성능 측정 원칙

- 변경 전후 동일 환경과 동일 입력으로 비교한다.
- 최소 5회 반복 후 중앙값과 IQR을 기록한다.
- 중앙값 기준 10% 이상 악화되면 원인을 분석하고 수정 또는 승인 사유를 기록한다.
- 실제 병목이 확인되기 전에는 캐시, 병렬화, 특수 자료구조를 도입하지 않는다.
- 성능 PR이 아니거나 입력 규모가 작으면 성능 측정은 생략할 수 있으며 사유를 기록한다.
