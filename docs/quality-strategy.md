# 품질 및 성능 전략

## 1. 품질 목표

우선순위는 **답안 보존 > 제출 일관성 > 보안 > 접근성 > 성능 > 부가 기능** 순서로 둔다. 성능 향상을 위해 정확성이나 결과 일관성을 희생하지 않는다.

## 2. 테스트 피라미드

### 단위 테스트

- 응시 상태 전이와 금지 전이
- 마감 시각 계산과 서버 시간 판정
- 문제 유형별 답안 검증
- 리비전 비교와 충돌 판정
- 제출 가능 여부와 자동 제출 원인

### 통합 테스트

- 답안 조건부 갱신과 트랜잭션 롤백
- 동일 멱등성 키 재요청
- 병렬 제출 요청에서 제출물 하나만 생성
- 권한·소유권·시험 상태 검사
- 마감 시각과 저장 요청의 경쟁 조건
- 감사 이벤트와 업무 데이터의 원자적 기록

### E2E 테스트

- 정상 시작, 자동 저장, 최종 제출
- 새로고침 후 답안과 남은 시간 복구
- 오프라인 변경 후 연결 복구
- 다른 탭·기기 충돌
- 마감 직전 저장과 제출
- 중복 제출 클릭
- 관리자 무효화 후 응시 화면 전환

## 3. 접근성 검증

- 키보드만으로 로그인, 문제 이동, 답안 선택, 제출 가능
- 포커스 순서와 포커스 표시가 명확함
- 저장·오프라인·오류·시간 경고가 스크린 리더에 전달됨
- 색상만으로 답안 여부나 오류를 구분하지 않음
- 확대 및 작은 화면에서도 문제와 조작 영역이 손실되지 않음
- 시간 연장 등 합리적 편의 제공 정책을 응시 세션에 반영 가능

## 4. 서비스 지표

운영 규모가 확정되면 수치를 승인하고 다음 지표를 대시보드로 관리한다.

| 지표 | 목표 방향 |
| --- | --- |
| 확인된 답안 유실 건수 | 0건 |
| 응시당 최종 제출물 중복 건수 | 0건 |
| 답안 저장 성공률 | 지속 측정 후 SLO 확정 |
| 최종 제출 성공률 | 지속 측정 후 SLO 확정 |
| 재접속 복구 성공률 | 지속 측정 후 SLO 확정 |
| 답안 저장·제출 지연시간 | p50, p95, p99 분리 관찰 |
| 자동 제출 복구 지연 | 만료 응시 적체와 함께 관찰 |

근거 없는 목표 숫자를 먼저 확정하지 않는다. MVP 부하 시험과 운영 환경 기준 측정 후 기준선을 설정한다.

## 5. 부하 시험 시나리오

1. **동시 입장:** 시험 시작 시 다수 사용자가 응시 세션과 문제를 조회한다.
2. **자동 저장 집중:** 사용자가 여러 문제를 변경하며 주기적으로 저장한다.
3. **마감 집중:** 종료 직전 저장과 제출 요청이 동시에 증가한다.
4. **재접속 파동:** 네트워크 장애 복구 후 상태 조회와 대기 답안 재전송이 집중된다.

입력 규모는 예상 운영 규모 `N`과 `10N`을 기본으로 하며, 대규모 위험이 있으면 `100N`까지 확장한다.

## 6. 성능 측정 원칙

- 변경 전후 동일 환경과 동일 데이터로 비교한다.
- 최소 5회, 권장 10회 반복한다.
- 중앙값과 IQR을 기록하고 필요하면 메모리·스루풋을 함께 측정한다.
- 중앙값 기준 10% 이상 악화되면 원인을 분석하고 수정 또는 승인 사유를 기록한다.
- 단순 p95 하나만으로 변경을 반려하지 않는다.
- 프로파일링 시 상위 병목 함수 3~5개 또는 Flamegraph를 첨부한다.
- 실제 병목이 확인되기 전에는 병렬화, 캐시, 특수 자료구조를 도입하지 않는다.

현재 단계는 실행 코드와 대표 부하 데이터가 없으므로 성능 측정을 수행하지 않는다. 문서 작성은 런타임 성능에 영향을 주지 않으며 가독성을 우선한다.

## 7. 관측성

- 모든 요청에 추적 ID를 부여한다.
- 로그는 구조화하고 사용자 식별자는 필요 시 가명화한다.
- 응시 시작, 답안 저장, 제출, 만료 작업의 성공·실패·지연을 메트릭으로 수집한다.
- 오류율, 제출 실패, 만료 적체, 데이터베이스 연결 고갈에 경보를 설정한다.
- 답안 원문, 토큰, 쿠키, 정답은 로그와 추적 데이터에 포함하지 않는다.

## 8. 출시 게이트

- 핵심 인수 조건 자동화 완료
- 답안 유실·중복 제출 경쟁 조건 테스트 통과
- 고위험 보안 이슈 없음
- 키보드 응시와 주요 접근성 흐름 검증
- 백업 복구 및 롤백 절차 점검
- 예상 부하 `N`에서 합의된 SLO 충족
- 운영 대시보드와 장애 대응 문서 준비
