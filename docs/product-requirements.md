# 제품 요구사항

## 1. 목적

시험 응시자가 제한된 시간 안에 문제를 풀고 답안을 안전하게 제출할 수 있으며, 관리자가 시험 진행 상태와 제출 결과를 신뢰할 수 있는 온라인 응시 도구를 제공한다.

## 2. 사용자 역할

| 역할 | 주요 책임 | 기본 권한 |
| --- | --- | --- |
| 응시자 | 배정된 시험 응시 및 답안 제출 | 자신의 시험·응시 기록만 조회 및 변경 |
| 시험 관리자 | 시험·문제·응시 대상·일정 관리 | 담당 시험 설정 및 진행 상태 조회 |
| 채점자 | 주관식 답안 검토 및 점수 확정 | 배정된 제출물의 채점 정보 조회·변경 |
| 시스템 운영자 | 장애 대응, 계정 및 권한 운영 | 업무상 필요한 최소 범위의 운영 권한 |

관리자, 채점자, 운영자 권한은 분리하며 동일 사용자가 여러 역할을 가질 때에도 각 요청에서 필요한 권한을 명시적으로 검사한다.

## 3. MVP 기능 범위

### 3.1 응시자 기능

- 인증 후 자신에게 배정된 시험 목록 조회
- 입장 가능 시간과 응시 상태 확인
- 시험 유의사항 확인 및 응시 시작
- 객관식 단일 선택, 객관식 복수 선택, 단답형 문제 응답
- 문제 이동, 답안 여부 표시, 임시 답안 자동 저장
- 서버 기준 남은 시간 확인
- 네트워크 복구 후 미전송 답안 재동기화
- 최종 제출 전 확인 및 제출 결과 조회

### 3.2 관리자 기능

- 시험 기본 정보, 일정, 제한 시간, 문제 구성 등록
- 응시자 배정 및 응시 가능 여부 설정
- 시험별 응시 상태와 제출 여부 조회
- 사유를 포함한 응시 무효화 또는 재응시 허용
- 감사 로그를 통한 주요 관리 작업 확인

### 3.3 MVP 제외 범위

- 웹캠·마이크 녹화 및 실시간 감독
- 화면 녹화, 브라우저 잠금, AI 부정행위 판정
- 서술형 자동 채점
- 다중 리전 액티브-액티브 운영
- 실측 근거가 없는 캐시, 메시지 큐, 마이크로서비스 분리

## 4. 시험 운영 정책

| 항목 | 기본 정책 | 관리자 설정 가능 여부 |
| --- | --- | --- |
| 입장 | 시작 시각부터 입장 가능 | 조기 입장 대기 화면만 허용 |
| 지각 입장 | 종료 시각 전이면 허용하되 종료 시각은 연장하지 않음 | 시험별 변경 가능 |
| 응시 시간 | `min(시작 시각 + 제한 시간, 시험 종료 시각)` | 시험별 제한 시간 설정 |
| 답안 수정 | 최종 제출 또는 만료 전까지 허용 | 문제 단위 잠금은 MVP 제외 |
| 새로고침 | 허용, 서버 저장 답안 복구 | 고정 |
| 일시 단절 | 허용, 로컬 대기열에 보관 후 재전송 | 고정 |
| 중복 로그인 | 마지막 활성 세션만 쓰기 허용, 기존 화면은 읽기 전용 전환 | 운영 정책으로 조정 가능 |
| 기기 변경 | 재인증 후 동일 응시 세션 복구 | 시험별 금지 가능 |
| 최종 제출 | 확인 절차 후 1회 확정, 이후 수정 불가 | 고정 |
| 시간 만료 | 서버의 마지막 저장 답안으로 자동 제출 | 고정 |

정책 변경은 이미 시작된 응시 세션에 소급 적용하지 않는다. 응시 시작 시 적용 정책의 스냅샷을 응시 세션에 저장한다.

## 5. 핵심 사용자 흐름

1. 응시자가 로그인한다.
2. 배정된 시험과 입장 가능 여부를 확인한다.
3. 유의사항 및 개인정보 처리 안내를 확인한다.
4. 서버가 응시 세션과 마감 시각을 생성한다.
5. 응시자가 문제를 풀고 답안을 자동 저장한다.
6. 연결 장애가 발생하면 변경 내용을 로컬 대기열에 보관한다.
7. 연결 복구 시 서버 버전을 확인하고 미전송 답안을 재전송한다.
8. 응시자가 최종 제출하거나 마감 시각에 서버가 자동 제출한다.
9. 서버가 제출 식별자와 제출 시각을 반환한다.

## 6. 기능 인수 조건

### 시험 시작

- **Given** 응시자에게 시험이 배정되고 입장 가능 시간인 경우
- **When** 응시자가 시험 시작을 요청하면
- **Then** 서버는 하나의 활성 응시 세션과 서버 기준 마감 시각을 반환한다.

### 자동 저장

- **Given** 진행 중인 유효한 응시 세션이 있는 경우
- **When** 응시자가 답안을 변경하면
- **Then** 화면에 즉시 반영되고 서버 저장 상태가 표시되며 동일 요청의 재전송은 중복 변경을 만들지 않는다.

### 재접속

- **Given** 저장된 답안이 있고 응시 시간이 남아 있는 경우
- **When** 응시자가 새로고침하거나 재인증하면
- **Then** 서버 저장 답안과 남은 시간이 복원된다.

### 최종 제출

- **Given** 제출되지 않은 진행 중 세션인 경우
- **When** 응시자가 최종 제출을 확인하면
- **Then** 모든 수락된 답안을 하나의 제출물로 확정하고 이후 답안 변경을 거부한다.

### 자동 제출

- **Given** 응시 세션의 마감 시각이 지난 경우
- **When** 저장 또는 상태 조회 요청이 도착하거나 만료 작업이 실행되면
- **Then** 서버는 마지막 저장 답안으로 정확히 한 번 제출을 확정한다.

## 7. 비기능 요구사항

- 답안 유실과 중복 최종 제출을 허용하지 않는다.
- 사용자에게 저장 상태, 연결 상태, 제출 상태를 숨기지 않는다.
- 키보드만으로 모든 응시 기능을 사용할 수 있어야 한다.
- 개인정보와 답안 데이터는 목적에 필요한 최소 범위로 수집한다.
- 핵심 변경은 사용자, 응시 세션, 시각, 사유를 포함한 감사 이벤트를 남긴다.
- 목표 동시 사용자 수와 지연시간은 운영 가정이 확정된 후 품질 전략의 기준값으로 관리한다.
