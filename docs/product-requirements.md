# 제품 요구사항

## 1. 목적

응시자가 온라인 연결 없이 준비된 시험지를 열어 제한 시간 안에 문제를 풀고, 제출 즉시 일관된 기준으로 평가·채점 결과를 확인할 수 있는 단순 응시 도구를 제공한다.

## 2. 사용자 역할

| 역할 | 주요 책임 | 기본 권한 |
| --- | --- | --- |
| 응시자 | 시험지 열기, 답안 작성, 제출, 결과 확인 | 현재 열린 시험과 자신의 응답만 조작 |
| 시험 준비자 | 시험지, 정답, 배점, 해설 작성 | 시험지 파일 생성·수정 |
| 채점 검토자 | 결과 파일 확인 및 필요 시 수동 검토 | 제출 결과와 문항별 채점 내역 조회 |

온라인 계정, 중앙 관리자, 시스템 운영자 역할은 MVP에서 두지 않는다. 동일한 사람이 시험 준비자와 채점 검토자를 겸할 수 있으나, 응시 화면에서는 제출 전 정답과 채점 기준을 표시하지 않는다.

## 3. MVP 기능 범위

### 3.1 응시자 기능

- 로컬 시험지 파일 불러오기
- 시험 제목, 유의사항, 제한 시간 확인
- 객관식 단일 선택, 객관식 복수 선택, 단답형 문제 응답
- 문제 이동, 답안 여부 표시, 제출 전 미응답 문항 확인
- 로컬 기준 남은 시간 표시
- 최종 제출 전 확인
- 제출 즉시 총점, 문항별 정오, 획득 점수, 필요 시 해설 확인
- 제출 결과를 시험별 누적 CSV에 기록하고, 사용자가 지정한 디렉터리의 응시자별 CSV를 저장 창 없이 갱신하거나 인쇄
- 홈·리포트·변환 화면으로 이동해도 다른 시험지를 불러오기 전까지 현재 시험지, 문항 위치, 작성 답안 유지

### 3.2 시험 준비·검토 기능

- 시험 기본 정보, 제한 시간, 문제 구성, 정답, 배점 작성
- 시험지 파일 구조 검증
- 자동 채점 결과 확인
- 단답형처럼 규칙 채점이 어려운 문항에 대해 검토 필요 표시
- 결과 파일의 응시자 식별 정보 포함 여부 선택

### 3.3 MVP 제외 범위

- 온라인 인증과 계정 관리
- 서버 저장, 자동 동기화, 재접속 복구
- 관리자 대시보드와 실시간 응시 현황
- 웹캠·마이크 녹화 및 실시간 감독
- 화면 녹화, 브라우저 잠금, AI 부정행위 판정
- 서술형 자동 채점과 복잡한 자연어 평가
- 다중 사용자 동시 운영, 큐, 캐시, 마이크로서비스 분리

## 4. 시험 운영 정책

| 항목 | 기본 정책 | 설정 가능 여부 |
| --- | --- | --- |
| 입장 | 시험지 파일을 열면 응시 가능 | 파일 배포 방식은 운영자가 별도 관리 |
| 응시 시간 | 시험지에 정의된 제한 시간을 로컬 타이머로 표시 | 시험별 설정 가능 |
| 답안 수정 | 제출 전까지 허용 | 고정 |
| 임시 저장 | 다른 시험지를 불러오기 전까지 메모리 보관 | 브라우저 종료 후 복구는 제외 |
| 최종 제출 | 제출 버튼 또는 시간 만료 시 현재 답안으로 채점 | 고정 |
| 시간 만료 | 만료 안내 후 현재 답안 제출을 유도하거나 자동 제출 | 시험별 설정 가능 |
| 결과 공개 | 제출 직후 점수와 문항별 결과 표시 | 해설 공개 여부 설정 가능 |
| 결과 보관 | 시험 ID·버전별 원본 이력 누적 및 지정 디렉터리의 사번별 CSV 갱신 | 최신·최고·전체 집계 전환, CSV 백업 후 삭제 가능 |

로컬 타이머는 단순 응시 편의를 위한 기능이다. 엄격한 감독과 중앙 통제가 필요한 시험은 본 MVP 범위가 아니라 별도 제품 정책으로 다룬다.

## 5. 채점 정책

- 객관식 단일 선택은 선택한 보기 ID가 정답과 같으면 배점을 부여한다.
- 객관식 복수 선택은 정답 집합과 응답 집합이 완전히 같을 때 배점을 부여한다.
- 단답형은 공백 정규화, 대소문자 무시 등 시험지에 정의된 단순 규칙만 적용한다.
- 부분 점수, 패널티, 복수 정답은 시험지 스키마에 명시된 경우에만 적용한다.
- 자동 판정이 어려운 문항은 `검토 필요`로 표시하고 총점에는 정책에 따라 0점 또는 보류 점수로 반영한다.

## 6. 인수 조건

### 시험지 불러오기

- **Given** 유효한 시험지 파일이 있는 경우
- **When** 응시자가 파일을 선택하면
- **Then** 시험 제목, 유의사항, 문제 목록, 제한 시간이 표시된다.

### 답안 작성

- **Given** 시험이 진행 중인 경우
- **When** 응시자가 문항에 답하면
- **Then** 답안 상태가 즉시 화면에 반영되고 문제 목록에 응답 여부가 표시된다.

### 최종 제출과 채점

- **Given** 응시자가 제출을 확인한 경우
- **When** 제출 처리가 실행되면
- **Then** 현재 답안 스냅샷을 기준으로 각 문항을 평가하고 총점을 계산한다.

### 결과 확인

- **Given** 채점이 완료된 경우
- **When** 결과 화면이 표시되면
- **Then** 총점, 만점, 문항별 정오, 획득 점수, 공개 가능한 해설이 표시된다.

### 시험 상태 유지

- **Given** 시험지를 불러와 응시 준비 또는 응답 중인 경우
- **When** 사용자가 홈, 리포트 또는 문항 변환 화면으로 이동했다가 현재 시험으로 돌아오면
- **Then** 다른 시험지를 불러오지 않은 한 동일한 시험지, 현재 문항 위치, 작성 답안이 유지된다.

### 응시자별 CSV 자동 저장

- **Given** 사용자가 기록 디렉터리와 쓰기 권한을 한 번 지정한 경우
- **When** 응시자가 시험을 제출하면
- **Then** 추가 저장 창을 띄우지 않고 해당 사번의 시험 ID·버전별 누적 CSV를 생성하거나 갱신한다.
- **And** 디렉터리 API 또는 권한을 사용할 수 없으면 브라우저 내부 누적 저장과 수동 CSV 저장을 계속 제공한다.

### 잘못된 시험지

- **Given** 필수 필드나 정답 기준이 누락된 시험지 파일인 경우
- **When** 응시자가 파일을 선택하면
- **Then** 응시를 시작하지 않고 수정 가능한 오류 메시지를 표시한다.

## 7. 비기능 요구사항

- 온라인 연결이 없어도 응시, 제출, 평가, 채점이 완료되어야 한다.
- 제출 이후 채점 결과는 동일 입력에 대해 항상 동일해야 한다.
- 응시 화면은 제출 전 정답과 채점 기준을 노출하지 않는다.
- 키보드만으로 모든 응시 기능을 사용할 수 있어야 한다.
- 개인정보와 결과 데이터는 목적에 필요한 최소 범위로만 파일에 포함한다.
- 대표 시험지 규모에서 화면 반응성이 저하되지 않아야 하며, 병목이 측정된 경우에만 최적화한다.
