# 제품 요구사항

## 1. 목적

QQQ 시험·인증 툴은 온라인 연결 없이 검사원 자격 인증에 필요한 평가와 증빙을 수행하고, SQLite 원본 결과를 기준으로 자격 판정과 인증서 발행을 통제하는 로컬 관리 시스템을 목표로 한다.

최종 목표는 시험 점수 오기재, 평가 항목 오기재, Gage R&R 분석 결과 누락, 교육 이수 근거 미확인, 인증서와 원본 자료 불일치를 시스템적으로 방지하는 것이다.

현재 구현은 필기시험 중심 UI와 SQLite 기반 R&R, 교육 검증, 통합 자격 판정, 인증서 이력 API를 포함한다. 문서의 요구사항은 이 구현을 유지·확장할 때의 기준이다.

## 2. 제품 범위

### 포함 범위

- 필기시험 JSON 로드와 기본 시험 실행
- 객관식 단일 선택, 객관식 복수 선택, 단답형 응시
- 필기시험 자동 채점과 결과 화면
- 기존 필기시험 결과 저장, CSV 리포트, 인증서 PNG 발행 흐름
- 로컬 실행과 자동 테스트
- 응시자와 자격 유형 관리
- 자격 유형별 필수 평가 계획 관리
- 필기시험(`written_exam`)
- 계수형 Gage R&R(`attribute_rr`)
- 계량형 Gage R&R(`variable_rr`)
- 교육 이수 검증(`training_verification`)
- 자격 판정과 승인 이력
- 정식 인증서 발행, 취소, 재발행 이력
- SQLite 원본 저장과 로컬 조회
- CSV 내보내기, 백업, 외부 제출용 보조 기능

### 제외 범위

- 온라인 서버, 계정 시스템, 자동 동기화
- 실시간 감독, 웹캠, 화면 녹화, 브라우저 잠금
- AI 이미지 판정
- 사진 판정 독립 시험 또는 `image_judgment` 평가 유형
- 측정되지 않은 병목을 전제로 한 캐시, 병렬화, 대형 구조 변경

## 3. 사용자 역할

| 역할 | 주요 책임 |
| --- | --- |
| 응시자/검사원 | 필기시험 응시, R&R 판정·측정 수행, 교육 증빙 제출 |
| 시험·평가 준비자 | 시험지, R&R 세트, 측정 계획, 교육 기준, 판정 기준 관리 |
| 검토자/승인자 | 산출 결과와 증빙 검토, 자격 판정 승인 |
| 로컬 운영자 | SQLite DB 파일, 백업, CSV 내보내기, 인증서 파일 보관 관리 |

온라인 계정은 두지 않는다. 역할은 로컬 운영 절차상의 책임 구분이며, 현재 구현은 계정 기반 권한 모델을 제공하지 않는다.

## 4. 핵심 원칙

1. SQLite를 원본 저장소로 사용한다.
2. CSV는 원본 저장소가 아니라 내보내기, 백업, 외부 제출용 보조 기능으로만 사용한다.
3. 제출 결과는 `attemptId` 또는 평가 원본 ID 기준으로 덮어쓰지 않는다.
4. 시험 내용, 배점, 합격 기준, 평가 기준이 변경되면 `revision`을 증가시킨다.
5. 제출 시점의 답안, 판정값, 측정값은 스냅샷으로 고정한다.
6. 제출 또는 확정 이후 원본 결과는 수정할 수 없다.
7. 정식 인증서는 시스템에서 확정된 결과만 사용하여 발행한다.
8. 필수 평가 또는 필수 증빙이 누락된 경우 정식 인증서 발행을 차단한다.
9. DB 장애 시 생성되는 `LOCAL_ONLY` 출력물은 정식 인증서가 아니며 DB 복구 후 재발행해야 한다.
10. 정답, 기준값, 채점 규칙은 제출 전 응시 화면에 노출하지 않는다.
11. 로그에는 답안 원문, 정답, 기준값, 개인정보를 기록하지 않는다.

## 5. 최종 지원 평가 유형

지원 평가 유형은 다음 네 가지로 고정한다.

```text
written_exam
attribute_rr
variable_rr
training_verification
```

### 5.1 written_exam

검사 기준, 작업 절차, 부적합 판정 기준, ESD, AOI, 외관검사, 수입검사, 공정검사 등 기본 지식을 확인한다. 객관식 단일 선택, 객관식 복수 선택, 단답형을 지원한다.

산출 결과는 총점, 문항별 정오, 문항별 획득 점수, 항목별 과락 여부, 합격 여부, 등급이다.

### 5.2 attribute_rr

계수형 Gage R&R이다. 사진 기반 판정은 별도 평가 유형이 아니라 `attribute_rr`의 `sampleMode=image`로 처리한다. 실물 샘플 판정은 `sampleMode=physical_sample`, 혼합 세트는 `sampleMode=mixed`로 관리한다.

산출 결과는 기준값 대비 전체 일치율, 반복 일치율, 종합 검출력, 정상품 일치율, 불량품 검출률, 1종 오류율, 2종 오류율, 불량 유형 일치율, 최종 판정이다.

기본 오류 정의는 다음과 같다.

```text
1종 오류: 기준 OK 샘플을 NG로 판정
2종 오류: 기준 NG 샘플을 OK로 판정
```

2종 오류는 고객 유출 위험이 있으므로 자격 판정에서 가장 엄격하게 관리한다.

### 5.3 variable_rr

계량형 Gage R&R이다. 수입검사, 공정 계측, 출하검사 등 수치 측정값으로 적합 여부를 판단하는 검사자의 측정 반복성, 재현성, 계측기 사용 적합성을 확인한다.

산출 결과는 EV, AV, GRR, %GRR, Part Variation, Total Variation, ndc, 공차 대비 측정 시스템 변동, 최종 판정이다. `studyPurpose`는 `process_msa`와 `inspector_qualification`을 구분하며 이번 리뉴얼의 기본 목적은 `inspector_qualification`이다.

초기 계산 방식은 구현 난이도가 낮은 `range` 방식을 우선 문서화하고, `anova`는 후속 확장으로 둔다.

### 5.4 training_verification

필수 교육 이수 여부, 최소 교육 시간 충족 여부, 교육 이수 증빙을 확인한다. 교육 시간은 점수 항목이 아니라 필수 충족 항목으로 관리한다.

## 6. 자격 유형별 필수 평가 구성

| 자격 유형 | written_exam | attribute_rr | variable_rr | training_verification |
| --- | --- | --- | --- | --- |
| 외관검사자 | 필수 | 필수 | 선택 | 필수 |
| AOI 판정자 | 필수 | 필수 | 선택 | 필수 |
| 수입검사자 | 필수 | 업무별 적용 | 계측기 사용 시 필수 | 필수 |
| 공정 계측 검사자 | 필수 | 선택 | 필수 | 필수 |
| 출하검사자 | 필수 | 업무별 적용 | 계측기 사용 시 필수 | 필수 |

시스템은 자격 유형별 필수 평가가 누락된 경우 정식 인증서 발행을 차단해야 한다.

## 7. 주요 기능 요구사항

### 7.1 필기시험

- 기존 필기시험 기능을 유지한다.
- 제출 전 정답과 채점 기준을 노출하지 않는다.
- 제출 후 답안은 수정할 수 없다.
- 채점 결과는 SQLite 원본 저장소에 저장한다.
- SQLite 전환 전 기존 CSV/브라우저 저장 기반 이력은 호환 계층으로만 유지한다.

### 7.2 계수형 Gage R&R

- 샘플은 `image`, `physical_sample`, `mixed`로 구성할 수 있어야 한다.
- 사진 파일 경로와 SHA-256 해시를 저장한다.
- 기준 OK/NG, 불량 유형, 불량 위치, 난이도, 기준 설명을 저장한다.
- 응시 화면에서는 기준값과 기준 설명을 노출하지 않는다.
- 동일 샘플 반복 제시를 지원하되 반복 여부는 숨긴다.
- 기준값 대비 일치율, 반복 일치율, 1종 오류율, 2종 오류율을 자동 산출한다.
- 결과가 기준 미달이면 정식 인증서 발행을 차단한다.

### 7.3 계량형 Gage R&R

- 측정 항목, 단위, 계측기명, 계측기 관리번호, 교정 유효기간을 저장한다.
- USL, LSL, 기준값, 부품 수, 반복 수를 설정한다.
- 측정값은 화면 입력과 CSV import를 모두 지원하되 확정 원본은 SQLite에 저장한다.
- EV, AV, GRR, %GRR, ndc를 계산한다.
- 결과가 기준 미달이면 정식 인증서 발행을 차단한다.

### 7.4 교육 이수

- 교육명, 이수 일자, 이수 시간, 증빙 경로를 저장한다.
- 필수 교육과 최소 교육 시간을 판정한다.
- 교육 증빙이 없으면 정식 인증서 발행을 차단한다.

### 7.5 자격 판정과 인증서

`validateCertificationReadiness()`는 `assessmentSessionId`, `qualificationTypeId`, `examineeId`를 입력받아 `decision`, `missingRequirements`, `blockingReasons`, `warnings`를 반환한다.

정식 인증서는 다음 조건을 모두 만족할 때만 발행한다.

1. 필수 필기시험 결과가 존재하고 합격 기준과 과락 기준을 만족한다.
2. 필수 계수형 R&R 결과가 존재하고 기준을 만족한다.
3. 필수 계량형 R&R 결과가 존재하고 기준을 만족한다.
4. 필수 교육 기록, 시간, 증빙이 기준을 만족한다.
5. 검토자 승인이 존재한다.
6. `certification_decision`이 `approved`이다.

정식 인증서에는 시스템 확정값만 포함하고, 점수·등급·평가 항목명은 수기 입력하지 않는다.

### 7.6 `LOCAL_ONLY` 출력물

DB 장애 등으로 SQLite 원본 저장과 정식 발행 이력이 불가능한 경우, 결과 확인용 `LOCAL_ONLY` 출력물을 만들 수 있다. 단, 이는 정식 인증서가 아니며 다음 조건을 반드시 지킨다.

- 파일명, 화면 문구, 출력물 본문에 `LOCAL_ONLY`와 중앙 조회 불가를 표시한다.
- 정식 인증서 ID, 정식 발행 이력, 승인 이력으로 취급하지 않는다.
- DB 복구 후 SQLite 원본 결과와 검토자 승인을 기준으로 정식 인증서를 재발행한다.
