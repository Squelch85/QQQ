# 보안 및 개인정보 기준

## 1. 보호 대상

- 시험 문제, 정답, 해설, 채점 기준
- 계수형 R&R 샘플 기준 OK/NG, 불량 유형, 기준 설명
- 계량형 R&R 기준값, 공차, 측정 원본
- 진행 중 답안, 제출 스냅샷, 판정값, 측정값
- SQLite 원본 DB와 백업 파일
- 정식 인증서 PNG, SHA-256 해시, 발행·취소·재발행 이력
- DB 장애 시 생성되는 `LOCAL_ONLY` 임시 출력물
- 성명, 사번, 부서 등 최소 개인정보

## 2. 보안 범위

이 제품은 네트워크, 계정, 서버를 사용하지 않는 로컬 자격 인증 평가 시스템이다. 서버 공격, 세션 탈취, API 인가, 온라인 동기화는 위협 범위에 포함하지 않는다. 대신 로컬 DB와 파일 노출, 제출 전 정답·기준값 표시, CSV 수식 인젝션, 인증서와 원본 불일치를 우선 통제한다.

## 3. 핵심 위협과 통제

| 위협 | 통제 |
| --- | --- |
| 제출 전 정답 또는 기준값 노출 | 응시 화면 모델에서 정답·채점 규칙·샘플 기준값 제외 |
| 사진 샘플 교체 | `image_hash`에 SHA-256 저장 |
| 결과 원본 변조 | SQLite 원본 저장, locked 필드, 취소·재평가 이력 사용 |
| 인증서 오기재 | DB 확정값만 사용하고 수기 점수·평가 항목명 입력 제거 |
| 필수 평가 누락 정식 인증서 발행 | `validateCertificationReadiness()` 차단 |
| `LOCAL_ONLY` 출력물의 정식 인증서 오인 | 파일명, 화면 문구, 인증서 본문에 임시 출력물임을 표시하고 중앙 조회 불가를 명시 |
| CSV 수식 인젝션 | export 시 위험 접두어 이스케이프 |
| 로그 개인정보 노출 | 답안 원문, 정답, 기준값, 개인정보 로그 금지 |
| 감사 로그 스냅샷의 민감 정보 포함 | 민감 필드 제거 후 저장하거나 원본 entity id와 해시·참조값만 기록 |
| 로컬 DB 무단 열람 | 운영체제 계정·파일 권한, 공유 PC 백업·삭제 절차 안내 |

## 4. 데이터 최소화와 보관

- 개인정보는 성명, 사번, 부서 등 업무상 필요한 최소 항목만 저장한다.
- 사번을 우선 식별자로 사용하되, 인증서 발행 전에는 정식 식별자를 요구한다.
- SQLite DB 기본 위치는 `data/exam_results.sqlite3`로 문서화한다.
- CSV는 보조 산출물이므로 원본으로 취급하지 않는다.
- 공유 PC에서는 업무 종료 후 DB 백업, CSV export 보관 위치 확인, 임시 파일 삭제 기준을 따른다.
- 보관 기간은 조직의 품질·개인정보 규정으로 정한다.

## 5. 로그와 감사 이력

감사 이력은 응시 생성, 제출, 채점, R&R 결과 계산, 교육 이수 확인, 자격 판정, 인증서 발행·취소·재발행, CSV 내보내기를 기록한다.

감사 로그는 다음 원칙을 따른다.

- 답안 원문, 정답, 기준값, 개인정보는 저장하지 않는다.
- 변경 전후 스냅샷이 필요하면 민감 필드를 제거한 `redactedBeforeJson`, `redactedAfterJson`만 저장한다.
- 재현과 추적에는 원본 entity id, action, actor, reason, createdAt, 결과 해시, 파일 경로 참조를 우선 사용한다.
- 인증서 발행 이력은 정식 인증서와 `LOCAL_ONLY` 출력물을 구분해 기록한다.

## 6. `LOCAL_ONLY` 출력물 경계

DB 장애 시 사용자가 결과 화면에서 내려받는 `LOCAL_ONLY` 출력물은 정식 인증서가 아니다. 이 출력물은 임시 확인용이며 다음 제한을 갖는다.

- `certification_decision=approved`에 기반한 정식 발행 이력이 아니다.
- 중앙 조회, 감사 이력, 취소·재발행 추적을 보장하지 않는다.
- DB 복구 후 SQLite 원본 결과와 검토자 승인을 기준으로 정식 인증서를 재발행해야 한다.

## 7. 부정행위 방지 한계

실시간 감독, 브라우저 잠금, 화면 녹화, 온라인 신원 확인은 제공하지 않는다. 사진 기반 평가는 AI 판정이 아니라 검사원이 직접 판정한 결과를 평가하는 기능이다.

## 8. 보안 검증

- 제출 전 UI 모델에 정답·기준값이 포함되지 않는 계약 테스트
- 사진 샘플 경로 조작과 해시 검증 테스트
- 제출 후 원본 수정 차단 테스트
- 정식 인증서 발행 차단 조건 테스트
- `LOCAL_ONLY` 출력물과 정식 인증서 구분 테스트
- CSV export 수식 인젝션 방지 테스트
- 로그의 답안·정답·기준값·개인정보 미포함 검증
