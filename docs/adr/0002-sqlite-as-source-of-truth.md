# ADR 0002: SQLite 원본 저장소 채택

- 상태: 승인
- 날짜: 2026-06-16

## 배경

검사원 자격 인증은 필기시험 점수뿐 아니라 계수형 R&R, 계량형 R&R, 교육 증빙, 자격 판정, 인증서 발행 이력이 서로 일치해야 한다. CSV나 브라우저 저장소를 원본으로 사용하면 결과 덮어쓰기, 인증서와 원본 불일치, 감사 이력 누락을 통제하기 어렵다.

## 결정

SQLite를 로컬 원본 저장소로 사용한다. 제출 답안, 판정값, 측정값, 산출 결과, 교육 이수, 자격 판정, 인증서 이력은 SQLite에 저장한다. CSV는 내보내기, 백업, 외부 제출, 계량형 측정값 import 편의 기능으로만 사용한다.

## 결과

인증서 발행은 SQLite에 확정된 결과만 참조한다. CSV 파일은 원본이 아니므로 CSV와 DB가 불일치할 때는 DB를 기준으로 판단한다. DB 장애 시 `LOCAL_ONLY` 인증서는 정식 인증서와 구분하고 중앙 조회가 불가능함을 표시한다.
