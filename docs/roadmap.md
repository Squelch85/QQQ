# 구현 로드맵

## 1단계: 시험지와 채점 규칙 확정

### 범위

- 지원 문제 유형과 답안 값 형식 확정
- 시험지 파일 스키마와 버전 규칙 작성
- 정답, 배점, 부분 점수, 단답형 정규화 정책 확정
- 응시·제출·채점 상태 전이 검토
- 결과 공개와 결과 파일 정책 확정
- 단일 실행 방식과 지원 환경 ADR 작성

### 완료 조건

- 모든 문제 유형에 입력·정답·채점 예시가 존재한다.
- 잘못된 시험지를 거부하는 검증 규칙이 명시된다.
- 동일 입력이 동일 점수를 생성하는 고정 테스트 벡터가 준비된다.
- 온라인 연결 없이 전체 기능을 실행한다는 범위가 승인된다.

## 2단계: 단순 응시 MVP 수직 슬라이스

### 범위

- 로컬 시험지 파일 불러오기
- 시험 시작과 로컬 타이머
- 문제 조회 및 세 가지 문제 유형 표시
- 메모리 기반 답안 입력과 응답 여부 표시
- 미응답 확인과 최종 제출
- 객관식·단답형 자동 채점과 결과 화면

### 완료 조건

- 네트워크 차단 상태의 정상 응시 E2E 시나리오가 통과한다.
- 제출 시 답안 스냅샷이 생성되고 이후 변경이 차단된다.
- 고정 테스트 벡터의 문항별 점수와 총점이 일치한다.
- 제출 전 응시 화면에 정답·채점 기준이 노출되지 않는다.

## 3단계: 오류 처리와 결과 보관

### 범위

- 시험지 스키마와 파일 크기 경계 검증
- 이해하기 쉬운 파일 오류 메시지
- 시간 만료 제출 정책
- 검토 필요 문항 처리
- 선택적 결과 파일 저장과 인쇄
- 개인정보 포함 여부 미리보기

### 완료 조건

- 손상되거나 잘못된 파일이 비정상 종료를 일으키지 않는다.
- 미응답, 시간 만료, 검토 필요 흐름의 E2E 테스트가 통과한다.
- 결과 파일에는 사용자가 선택하지 않은 개인정보가 포함되지 않는다.
- 파일 저장 실패 시 원래 채점 결과가 유지된다.

## 4단계: 사용성 및 접근성 안정화

### 범위

- 키보드 문제 이동과 포커스 관리
- 스크린 리더용 응답·시간·결과 안내
- 확대 및 작은 화면 레이아웃
- 시험지 작성자용 검증 도구 또는 템플릿
- 대표 운영체제 패키징과 실행 안내

### 완료 조건

- 키보드만으로 전체 응시와 결과 확인이 가능하다.
- 핵심 상태가 색상 이외의 텍스트와 의미 구조로 전달된다.
- 지원 환경에서 설치 또는 실행 절차가 재현된다.
- 예제 시험지가 스키마 검증과 전체 흐름 테스트를 통과한다.

## 5단계: 필요가 확인된 확장

### 후보 범위

- 명시적 로컬 임시 저장과 복구
- 암호화 또는 서명된 시험지 패키지
- 결과 파일 일괄 집계
- 수동 채점 전용 화면
- 추가 문제 유형과 채점 규칙
- 시험지 작성 편의 기능

### 진입 조건

- 실제 사용자 요구나 프로파일에서 명확한 필요와 병목이 확인된다.
- 도입 복잡도, 개인정보 영향, 예상 순이득을 문서화한다.
- 성능 변경은 최소 5회 측정한 중앙값과 IQR로 검증한다.
- 병렬화는 순이득이 20% 이상 예상되고 채점 결정성 검증 방법이 있을 때만 적용한다.
- 온라인 중앙 기능은 오프라인 단순성에 미치는 영향을 별도로 검토한다.

## 우선순위 원칙

1. 채점 정확성과 결과 결정성을 가장 먼저 보장한다.
2. 제출 시점의 답안을 고정하고 이후 변경을 막는다.
3. 온라인 의존성과 불필요한 운영 복잡성을 추가하지 않는다.
4. 접근성을 출시 후 보완 항목으로 미루지 않는다.
5. 측정되지 않은 성능 문제를 위한 복잡성을 추가하지 않는다.

## 주요 위험과 대응

| 위험 | 대응 |
| --- | --- |
| 시험지 스키마 변경으로 호환성 저하 | 명시적 버전과 마이그레이션 규칙 |
| 잘못된 정답·배점으로 오채점 | 시작 전 검증과 고정 테스트 벡터 |
| 제출 후 답안 변경 | 제출 스냅샷과 종결 상태 강제 |
| 단답형 규칙의 모호성 | 제한된 정규화 규칙과 검토 필요 처리 |
| 정답 파일 직접 열람 | 대상 사용 사례와 보안 한계 명시, 필요 시 패키지 보호 확장 |
| 결과 파일 개인정보 노출 | 기본 익명, 선택 입력, 저장 전 미리보기 |
| 과도한 초기 구조 | 로컬 단일 애플리케이션과 측정 기반 확장 |
