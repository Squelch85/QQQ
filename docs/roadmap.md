# 구현 로드맵

## 1단계: 정책 및 계약 확정

### 범위

- 사용자 역할과 권한 행렬 확정
- 시험 일정, 지각 입장, 재접속, 중복 로그인 정책 확정
- 도메인 상태 전이 검토
- API 명세와 오류 코드 작성
- 데이터 모델과 마이그레이션 초안 작성
- 기술 스택 및 운영 환경 ADR 작성

### 완료 조건

- 제품·개발·운영 담당자가 핵심 정책을 승인한다.
- 모든 응시 상태 전이에 인수 조건이 존재한다.
- 응시자 API에서 제외할 민감 필드가 명시된다.
- 예상 동시 응시자 수와 지원 브라우저가 결정된다.

## 2단계: 응시 MVP 수직 슬라이스

### 범위

- 인증과 배정 시험 조회
- 응시 시작과 서버 마감 시각
- 문제 조회 및 세 가지 문제 유형 표시
- 답안 자동 저장과 저장 상태 UI
- 최종 제출 및 제출 결과
- 기본 관리자 시험·문제·배정 기능

### 완료 조건

- 정상 응시 E2E 시나리오가 통과한다.
- 중복 저장과 중복 제출이 멱등하게 처리된다.
- 서버 시간 기준으로 마감이 판정된다.
- 정답·채점 기준 노출 계약 테스트가 통과한다.

## 3단계: 장애 내성과 일관성

### 범위

- 새로고침 및 재인증 복구
- 오프라인 대기열과 재동기화
- 다른 탭·기기 충돌 처리
- 자동 제출과 만료 응시 복구 작업
- 변경 불가능한 감사 이벤트
- 백업 복구 및 운영 런북

### 완료 조건

- 네트워크 단절·복구 E2E 시나리오가 통과한다.
- 마감과 저장·제출의 경쟁 조건 테스트가 통과한다.
- 장애 주입 후 답안 유실과 중복 제출이 발생하지 않는다.
- RPO·RTO 기준의 복구 훈련 결과를 기록한다.

## 4단계: 운영 및 채점 기능

### 범위

- 응시 현황 대시보드
- 권한과 사유를 포함한 무효화·재응시 처리
- 단답형 및 주관식 수동 채점 흐름
- 결과 공개 정책과 내보내기
- 운영 경보와 감사 조회

### 완료 조건

- 관리자 권한 분리 테스트가 통과한다.
- 모든 고위험 조치에 행위자와 사유가 기록된다.
- 내보내기 데이터의 개인정보 범위와 보존 정책이 승인된다.

## 5단계: 규모 및 경험 확장

### 후보 범위

- 문제·선택지 무작위화
- 편의 제공 및 접근성 고도화
- 대규모 시험 예약과 용량 사전 점검
- 채점 워크플로 확장
- 제한적인 캐시·큐·서비스 분리
- 정책 검토를 마친 선택적 감독 기능

### 진입 조건

- 운영 지표나 프로파일에서 명확한 요구와 병목이 확인된다.
- 도입 복잡도, 개인정보 영향, 예상 순이득을 문서화한다.
- 성능 변경은 최소 5회 측정한 중앙값과 IQR로 검증한다.
- 병렬화는 순이득이 20% 이상 예상되고 정확성 검증 방법이 있을 때만 적용한다.

## 우선순위 원칙

1. 답안 유실 위험을 가장 먼저 제거한다.
2. 최종 제출의 정확히 한 번 효과를 보장한다.
3. 보안과 개인정보 통제를 기능과 함께 구현한다.
4. 접근성을 출시 후 보완 항목으로 미루지 않는다.
5. 측정되지 않은 성능 문제를 위한 복잡성을 추가하지 않는다.

## 주요 위험과 대응

| 위험 | 대응 |
| --- | --- |
| 정책 변경으로 도메인 로직 재작업 | 구현 전 정책 승인, 응시별 정책 스냅샷 |
| 네트워크 장애로 답안 유실 | 자동 저장, 임시 대기열, 리비전 기반 재동기화 |
| 마감 시 제출 요청 폭증 | 부하 시험, 멱등 제출, 만료 복구 작업 |
| 정답 노출 | 응시자 DTO 분리와 계약 테스트 |
| 관리자 오조작 | 최소 권한, 확인 절차, 필수 사유, 감사 로그 |
| 과도한 초기 인프라 | 모듈형 모놀리스와 측정 기반 확장 |
